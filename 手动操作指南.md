# GitHub手动操作完整指南

## 第一步：克隆GitHub仓库到本地

1. 打开命令行（cmd）
2. 创建一个临时文件夹：
   ```
   cd D:\
   mkdir github_temp
   cd github_temp
   ```

3. 克隆您的GitHub仓库：
   ```
   git clone https://github.com/Vinsonhuang21Xin/score-counter.git
   ```

4. 进入克隆的文件夹：
   ```
   cd score-counter
   ```

## 第二步：复制项目文件

1. 打开文件资源管理器
2. 进入 `D:\github_temp\score-counter`（刚克隆的空仓库）
3. 打开另一个窗口，进入 `D:\Claude Code\002_ScoreCounterApp\001_project_files\score_counter`（您的项目）
4. 从项目文件夹复制**所有文件**（包括隐藏文件.gitignore）
5. 粘贴到 `D:\github_temp\score-counter`
6. 选择"替换目标中的文件"

## 第三步：提交并推送到GitHub

回到命令行（确保在 `D:\github_temp\score-counter` 目录）：

```
git add .
git commit -m "Flutter计分器App"
git push origin main
```

如果要求输入用户名和密码：
- **用户名**：Vinsonhuang21Xin
- **密码**：输入您的GitHub Personal Access Token（不是登录密码）

## 第四步：获取GitHub Personal Access Token（如果需要）

1. 访问：https://github.com/settings/tokens
2. 点击 "Generate new token" → "Generate new token (classic)"
3. 勾选权限：
   - ✓ repo（完整仓库访问权限）
   - ✓ workflow（GitHub Actions权限）
4. 点击 "Generate token"
5. 复制生成的token（只显示一次！）
6. 在git push时，用户名输入 `Vinsonhuang21Xin`，密码粘贴这个token

## 第五步：运行GitHub Actions构建APK

1. 推送成功后，访问：https://github.com/Vinsonhuang21Xin/score-counter/actions
2. 等待几秒，页面会自动刷新
3. 您会看到 "Build Android APK" 工作流
4. 点击右侧的 "Run workflow" 按钮
5. 选择 "main" 分支
6. 点击绿色的 "Run workflow" 按钮
7. 等待10-15分钟
8. 构建完成后，点击进入构建记录
9. 滚动到底部，在 "Artifacts" 区域
10. 点击 "apk" 下载APK文件

## 常见问题

### Q1: git push时提示"Authentication failed"
A: 需要使用Personal Access Token，不能用登录密码

### Q2: 克隆时提示"无法连接github.com"
A: 确保VPN已开启，在浏览器访问 https://github.com 确认可访问

### Q3: GitHub Actions没有自动运行
A: 需要手动点击 "Run workflow" 按钮

### Q4: 找不到下载APK的地方
A: 在Actions页面，点击构建记录，滚动到底部，找到 "Artifacts" 区域
